# 데이터베이스의 기본 기능 

## 데이터의 검색과 갱신(CRUD)

- 검색(Read)
  - 데이터베이스에서 가장 중요한 기능
  - 검색 = 참조 = 추출

- 갱신
  - **등록(Create)** : DB에 데이터를 생성합니다.
  - **수정(Update)** : DB에 데이터를 수정합니다.
  - **제거(Delete)** : DB에 데이터를 제거합니다.

CRUD의 기능들 중에서 하나라도 빠지면 데이터베이스의 역할을 못함

- 고유성(Uniqueness) : 중복되는게 없고 유일해야함
- 처리 성능 : 어느 정도의 빠르기로 검색 처리 가능한지

## 동시성 제어(= 베타 제어)

복수 사용자의 갱신을 조절하기 위한 기능

- 복수 사용자
1. A가 파일을 열고 있을 때 B는 **r/w 불가능**
2. A가 파일을 열고 있을 때 B는 **r만 가능** -> 읽기 전용(Read Only)
3. A가 파일을 열고 있을 때 B는 **r/w 가능** -> 나중에 수행된 쪽의 갱신이 반영

> 3과 같은 경우를 더티 쓰기(Dirty Write)라고 합니다.
<br>데이터 무결성 관점에서 기피합니다.

- **트레이드오프(trade off) 관계**
  - 어느 사용자에게 괜찮은 갱신 제어가 다른 사용자에게는 불편한 상황

```
ex) B입장에서는 1이 행위 제한이 가장 심하고, 3이 가장 느슨
하지만 A입장에서는 3은 마음대로 갱신되니 위험하고, 1은 방해를 안받아서 가장 좋음
```

괴로운 트레이드오프 관계 : A, B의 입장에서 중간 지점을 맞추지 어려움

## 장애 대응

데이터 소실 되더라도 복원이 가능해야 함

- 데이터 소실 문제의 대책
  - **데이터 다중화** : 데이터를 여러 장소에 분산해서 유지 
  <br>-> 예방책
  - **백업** : 데이터 소실 시 복원하는 방법 
  <br>-> 사후대책

- 데이터 소실의 원인
  - **트레이드오프의 딜레마** : 서비스 레벨과 비용 사이

## 보안

보존된 데이터를 숨기기 위한 기능

### 데이터베이스를 사용자에게 보이지 않게 설계하는 이유

1. 사용자는 서버를 의식할 필요 없음
- 사용자는 클라이언트만 조작하면 됨 데이터베이스 서버를 조작할 필요X
  <br>-> if 조작가능하면 **보안 구멍**

2. 데이터베이스 기밀성이 높다
- 기밀성이 높아서 일반에 공개할 수 없는 내용이 상당수 포함

# 데이터베이스의 종류

## 계층형 데이터베이스 

- 역사상 최초로 등록, 가장 오래된 형태의 데이터베이스
- 컴퓨터에 계층 구조로 저장되어 있는 폴더와 파일 형태
- 오래된 방식이지만 아직 이용되고 있음

## 관계형 데이터베이스

- 2차원 표 형식으로 **열(Column)**과 **행(row)**으로 설계된 데이터베이스
- 많은 데이터를 **키(key)**와 **관계**라는 연결 고리로 연결하여 데이터를 조회하고 가공하는 방식 
- 현재 **가장 많이 쓰이는 방식**

<img src="/images/관계형 데이터베이스.png" width="400" height="400"/>

## 객제지향 데이터베이스

- 객체형식으로 데이터를 관리하는 데이터베이스
- 자바(Java)와 C++ 같은 객체 지향 언어의 객체 지향 프로그래밍(object-oriented programming)에 적합한 데이터베이스

## NoSQL 데이터베이스

- NoSQL = Not only SQL
- 관계형 데이터베스의 일부 기능을 버려서 성능(처리 속도)을 높임
- 대량의 데이터를 **고속으로 처리하는데 주로 이용** -> **웹서비스**
